# PRD Template (Lenny Rachitsky Style)

A lightweight, clear PRD format. Good for features where you want to move fast and keep the doc focused.

---

## [Project Title]

**Description:** What is it?
[1â€“2 sentences describing the feature or initiative.]

**Problem:** What problem is this solving?
[Clear problem statement grounded in user pain.]

**Why:** How do we know this is a real problem and worth solving?
[Evidence: user interviews, data, support tickets, churn reasons.]

**Success:** How do we know if we've solved this problem?
[Specific metrics and qualitative indicators.]

**Audience:** Who are we building for?
[Target user segment(s) and personas.]

**What:** Roughly, what does this look like in the product?
[High-level description of the solution; include mockups or flows if available.]

**How:** What is the experiment plan?
[How you'll validate: A/B test, beta, phased rollout, etc.]

**When:** When does it ship and what are the milestones?
[Timeline, key dates, dependencies.]
